Задание 2.1
Получаем объявления по его идентификатору
Шаги:
Отправить GET /api/1/item/{id_item}
Ожидаемый результат: Все запросы возвращают 200 OK. Данные в ответах соответствуют отправленным.
Фактический результат: Статус FAILED 400

Позитивные сценарии (E2E)
Полный цикл жизни объявления (Создание -> Проверка -> Статистика)
Шаги:
Отправить POST /api/1/item с валидными данными. Сохранить id из ответа.
Отправить GET /api/1/item/{id}.
Ожидаемый результат: Все запросы возвращают 200 OK. Данные в ответах соответствуют отправленным.
Фактический результат: Статус FAILED Найдена ошибка 405/404.

Получение объявлений по идентификатору продавца
Шаги:
Отправить GET /api/1/seller/{sellerId}/item.
Ожидаемый результат: 200 OK. Список содержит все объявления данного продавца.
Фактический результат: Статус FAILED 404

Валидация экстремально больших числовых значений (Boundary Testing)
Шаги: 
Отправить POST /api/1/item с ценой 999999999999999999999.
Ожидаемый результат: 400 Bad Request.
Фактический результат: 200

Проверка устойчивости к SQL-инъекциям
Шаги: 
Отправить в поле name значение "achoo' OR 1=1;--".
Ожидаемый результат: 200.
Фактический результат: 400

Валидация отрицательных значений

Отправить в поле price значение -100.

Фактический результат: 200 OK.
